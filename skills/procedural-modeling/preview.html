procedural-modeling

Skill preview for code-first Houdini procedural modeling with the Edini harness workflow.

Harness-first rule: Use houdini_run_python_sandbox for initial procedural asset generation when available.

Workflow

  1. Create a recipe first - State asset type, backend, parameters, and expected structural checks.
  2. Choose backend - python_sop, vex_wrangle, or node_network.
  3. Use the harness first - Prefer houdini_run_python_sandbox over raw execution.
  4. Preserve failed nodes - Call houdini_collect_diagnostics before deleting failed procedural nodes.
  5. Verify structurally - Use houdini_verify_asset and/or houdini_inspect_geo.
  6. Capture safely - Use houdini_capture_viewport_safe. Do not explore Qt widgets or unsupported viewport internals.

Harness Rules

  • Do not use raw houdini_run_python for initial procedural asset generation when houdini_run_python_sandbox is available.
  • Do not delete a failed procedural node before houdini_collect_diagnostics.
  • Do not explore Qt widgets, main windows, viewport internals, or unsupported HOM APIs to capture images.
  • Diagnose failed Python SOP cooks before switching to manual node-network generation.
