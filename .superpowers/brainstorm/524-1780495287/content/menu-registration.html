Edini 菜单栏注册 — 当前状态 vs 目标

目前 Edini 只是一个浮动 QWidget 窗口，需要手动 Python 命令启动。我们要把它注册为 Houdini 原生 UI 的一部分。

当前启动方式

Houdini Python Shell 中手动执行
>>> from edini import createPanel
>>> panel = createPanel()
>>> panel.show()
# 弹出独立浮动窗口

Houdini 菜单栏注册有几种方式，你倾向于哪种？

A

顶部菜单栏 + 浮动面板

在 Houdini 主菜单栏添加"Edini"菜单项（与 File / Edit / Render 同级），点击打开可停靠面板。类似 Houdini 自带的 Python Shell、Parameter Editor 等。

最符合"Houdini 原生感"，可停靠到主窗口任意位置

B

Shelf 工具按钮 + 弹出窗口

在 Houdini 顶部 Shelf 栏添加一个 Edini 图标按钮，点击弹出对话窗口。类似 Houdini 的其他 Shelf 工具。

快捷方便，但窗口是浮动的，不能停靠

C

两者都要 — 菜单 + Shelf 双入口

菜单栏添加 Edini 入口（管理面板），Shelf 添加快捷按钮（快速对话）。给用户两种启动路径。

最灵活，但实现量稍大
