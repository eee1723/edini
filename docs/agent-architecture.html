┌──────────────────────────────────────────────────────────────────────────────┐ │ LAYER 0: Pi 内置系统提示词 │ │ • 工具定义（read, write, edit, bash, describe_image） │ │ • 核心智能体指令 │ │ • 思考级别指引 │ │ • 安全提示 │ ├──────────────────────────────────────────────────────────────────────────────┤ │ LAYER 1: AGENTS.md / CLAUDE.md（自动发现） │ │ • 从 ~/.pi/agent/AGENTS.md 加载的项目级指令 │ │ • 从当前目录向上遍历各级父目录的 AGENTS.md 文件 │ │ • 目前在 Edini 项目中未配置（可根据需要添加） │ ├──────────────────────────────────────────────────────────────────────────────┤ │ LAYER 2: ⚠️ 参考图检测指令 ← 程序化钩子 │ │ • 仅在 event.images.length > 0 时注入 │ │ • "用户附带了 N 张参考图。你必须：..." │ │ • 5 步强制验证流程 │ │ • 不依赖 LLM 自主判断 —— 代码强制执行 │ ├──────────────────────────────────────────────────────────────────────────────┤ │ LAYER 3: Edini Houdini 上下文 ← edini-context 扩展 │ │ • 环境身份："你正在 Houdini 21 中运行，是 Edini 助手" │ │ • 工具能力概览 │ │ • 交互方式说明（用户通过面板与你交流，可看到视口） │ ├──────────────────────────────────────────────────────────────────────────────┤ │ LAYER 4: 操作指南 1-7 │ │ • 先搜索后创建、布局节点、优先专用工具、告知路径、错误处理、默认路径 /obj、 │ │ 获取场景上下文 │ ├──────────────────────────────────────────────────────────────────────────────┤ │ LAYER 5: 视觉验证决策框架 │ │ • 🔴 必须截图 / 🟡 建议截图 / 🟢 跳过截图的分类表格 │ │ • 6 步视觉任务工作流程（创建→截图→描述→对比→修正→完成） │ ├──────────────────────────────────────────────────────────────────────────────┤ │ LAYER 6: 铁律（知识库） ← ~/.pi/agent/edini-knowledge/rules.json │ │ • 用户自行定义的规则，按分类（避坑、技巧、工作流、配置）组织 │ │ • 仅启用状态的规则被注入 │ │ • 若无规则则为空 │ └──────────────────────────────────────────────────────────────────────────────┘
◉ 程序化保障： Layer 2（参考图检测指令）由 before_agent_start 钩子在检测到 event.images.length > 0 时动态注入系统提示词的最前面，不依赖 LLM 的自主判断。

◉ 持久性： Layers 3-6 在每次 LLM 调用前都会通过 before_agent_start 事件重新注入。 即使发生上下文压缩（compaction），这些钩子始终运行，核心指令不会丢失。

7. 视觉管线（pi-visionizer）

7.1 两种运行模式

模式触发条件流程使用场景
